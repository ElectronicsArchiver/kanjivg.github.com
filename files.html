KanjiVG files

Coverage

KanjiVG contains vector graphics which represent the Japanese versions of the Chinese characters (henceforth "kanji") and other characters of the Japanese JIS standard encodings known as "level one" and "level two".

The vector graphics are based on Japanese schoolbook fonts The formats of the kanji are those of Japan rather than China, thus the stroke shapes and the stroke order follow the Japanese example in all cases. Please see our reference page for exact details of the fonts and other references used as models.

Since the data was made public around 2008, some additions have been made to the data, such that it now extends to a few characters which do not belong to the level one or two JIS standards. However, all the additions still follow Japanese standards.

Other open source projects exist which cover traditional (Taiwan/Hong Kong) and simplified (mainland China and Singapore) Chinese characters. Please see the "related projects" link at the left of this page.

Format

The KanjiVG file format is an extension of SVG. All KanjiVG files are 100% SVG-compliant, and can be opened by any SVG viewer or editor as-is. The shape of each kanji stroke (an individual line) is described as an SVG path on an area of size 109x109 pixels. Since these are vector graphics, they can be rescaled at will to any convenient size.

The SVG file is organized to reflect both the structure and stroke order of the kanji. Furthermore, several additional attributes provide extra information about the kanji's structure. See SVG Format for details.

Character models

The main graphical data is is based on schoolbook fonts (教科書体きょうかしょたい), which is a variation of the calligraphic kaisho style (楷書かいしょ). The exact fonts used as models for the KanjiVG graphical data are listed on the References page.

There is data for variations of characters and stroke order. When characters in schoolbook style deviate from traditional kaisho, there might be an extra version for kaisho, labelled with the suffix -Kaisho. There are also different versions for different stroke orders. See Variants for more information on the file naming and what variants exist.

Individual kanji files

In the repository for KanjiVG, each kanji has at least one file which is given a name of the lower-case hexadecimal Unicode value of the kanji, padded to five characters with zeros, followed by the suffix .svg.

Download files

The download XML files are created from the individual kanji files, excluding the variant forms. The Python scripts to create the files are in the repository.
