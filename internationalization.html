Limitation of Kanjivg

KanjiVG currently focuses only on Japanese format. The stroke order of traditional Chinese used in Taiwan and Hong Kong, simplified Chinese used in mainland China and Singapore, and the Korean and Vietnamese variants, may differ.

Multi-country approach

The need is, yes, to focus on Japanese stroke order first. This Japanese set will create the core of the multi-countries approach.

Then, if for a given country out of Japan, there is an official statement for:

  • a different shapes
  • a different stroke order

Then a new page should/vectorial description should be created.

For example: 王

  • Kanji / Japanese: 王 = H-S-H-H
  • China: 王 = H-H-S-H
  • Taiwan: = China

This DIFFERENCE request the creation of ONE file, which *may* *be* *named* "王>KanjiC" (KanjiC for China).

Taiwan will use Taiwan file 王, if no available then use China file 王, if no available then use Japan file 王.

Conventions (Beta)

Commons SOP

Multi-countries approach on Wikimedia Commons' Stroke Order Project use the following prefixes:

  • J - stand for Japanese 'official' stroke order
  • T - stand for Taiwanese (ROC) official stroke order
  • (nothing) or M or C - stand for Modern China (PRC)
  • H - may stand for Hongkong
  • K - may stand for Korea
  • S - may stand for Singapore

J (Japan), T (ROC), M (PRC), H (Hongkong) have official or semi official standards (see below) K (Korea), S (Singapour), have no official stroke order standard.
